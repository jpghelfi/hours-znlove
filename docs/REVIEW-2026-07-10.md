# Expert Review — Hours Tracker (2026-07-10)

Three independent reviews: UI/UX, Backend/Security, QA (functional, against a live instance
with real API writes — all test data cleaned up and verified).

**Overall verdicts:** Backend: "core authz is implemented correctly; risks are operational
edge-cases, all cheap to fix." UI: "tight, coherent internal app; fix the top 4 and it's in
solid shape." QA: happy paths all PASS; validation and error-handling paths FAIL.

## P0 — fix before wide team use

| # | Finding | Source |
|---|---------|--------|
| 1 | `SESSION_SECRET` falls back to a hardcoded string → forgeable admin cookies if env var ever missing. Hard-fail at startup instead. | Backend |
| 2 | Project membership not enforced on writes — `/entry` & `/api/cell` accept any `project_id` (picker-only filtering). | Backend |
| 3 | Unhandled 500s: `/entry` with bad date; `/week?monday=banana`; `/schedule?start=banana`; `/reports?date_from=banana`. Guard all date parsing → 400/fallback. | QA D1, D2 |
| 4 | Hours accept negative / 999999 / NaN on every write path → corrupt totals & capacity. Validate `0 < hours ≤ 24` server-side. | QA D5, D6; Backend |

## P2 — polish / hardening

- Raw Notion error bodies leak to clients (`/api/cell` 400s include integration name). Sanitize. (QA D9, Backend)
- `/entry` with `hours=0` creates a real 0h row (inconsistent with cell delete semantics). (QA D7)
- `date_from` without `date_to` silently ignored on /reports. (QA D8)
- Reports scope toggle & Schedule pivot toggle drop active filters. (UI 6, 7)
- Timer invisible from other pages; Stop doesn't signal "still needs Save". (UI 5)
- Client-added grid rows vanish on reload until first save; `newPair()` lacks duplicate guard. (UI 8, 9)
- Mobile: cells lack touch affordance; add `inputmode="decimal"` for iOS. (UI 10, 11)
- Sticky first column wrong background on schedule group rows; "W07/06" MM/DD ambiguity. (UI 12, 13)
- Sessions irrevocable for 30 days (allowlist checked only at login); re-check per request. (Backend)
- Pin `notion-client>=3,<4` + lock deps; cache `_project_name_map()`; add Person filter to week queries (perf). (Backend)
- CSV formula injection in exports (`=…` descriptions execute in Excel). Prefix-quote. (Backend)
- Self-host Poppins (Google Fonts CDN dependency). (UI 15)

## QA pass/fail snapshot
Happy paths (all GET pages, presets, CSV, cell/allocation/entry create-read-delete): **PASS**.
Failures concentrated in: malformed date params (500s), hours validation, non-Monday weeks,
duplicate-entry consolidation, partial custom ranges.
